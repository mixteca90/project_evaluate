# 조별 프로젝트 평가 앱 — 기능 맵

**근거 문서**: [`조별_프로젝트_평가기준.md`](./조별_프로젝트_평가기준.md), [`앱_페이지구성_내비게이션맵.md`](./앱_페이지구성_내비게이션맵.md)
**목적**: 페이지 구성안의 각 화면이 실제로 수행해야 할 기능을 모듈 단위로 정리하고, 특히 복잡도가 높은 **점수 산출 로직**을 상세화한다.

---

## 1. 기능 모듈 분류

| 모듈 ID | 모듈명 | 설명 | 관련 페이지 |
|---|---|---|---|
| M1 | 신원확인/역할판별 | 이름 선택 → 소속 조·역할(학생/강사) 자동 결정 | P1 |
| M2 | 평가 대상 조회 | 역할에 따라 평가 가능한 조 목록 필터링(자기 조 제외) + 진행 배지 | P2 |
| M3 | 조 정보·자기신고 조회 | 조 기본정보 + 5-1 자기신고(완료범위/향후계획/AI협업범위) 표시 | P3 |
| M4 | 세부항목 채점 입력 | 13개 세부항목 선택형 채점, 실시간 합계 표시, 선택 즉시 자동 임시저장 | P3 |
| M5 | 의견 입력·제출 관리 | 의견 텍스트 저장 + 제출완료 상태 전환, 마감 전까지 재수정 가능한 제출 상태 관리 | P4 |
| M6 | 점수 집계 엔진 | 절사(최고·최저 2인 제외) → 강사 가중(×3) → 평균 산출 (조별·세부항목별), 마감 전 잠정치 / 마감 후 확정치 구분 | 백엔드 (전 페이지 공통) |
| M7 | 전체 결과 조회 | 조별 최종 총점 순위 리스트 | P5 |
| M8 | 조별 상세 결과 조회 | 조별 세부항목 확정 점수 + 의견 목록(익명) | P6 |
| M9 | 강사 관리 기능 | 강사(이정현) 비밀번호 인증, 평가자별 원점수·절사여부 매트릭스 조회, 평가 마감 처리 (강사 본인 채점은 M2~M5를 통해 학생과 동일하게 처리) | I1 |
| M10 | 진행상태 관리 | 조별/전체 제출 현황(N/M명) 실시간 갱신 | P2, P5 |

## 2. 기능 상세 명세

| 기능ID | 기능명 | 트리거 | 입력 | 처리 로직 | 출력 | 관련 평가기준 |
|---|---|---|---|---|---|---|
| F1 | 본인 확인 | P1 이름 선택 | 사용자 선택(이름 또는 "강사") | 명단 조회로 role(student/instructor), group 매핑. **강사 선택 시 비밀번호 검증 단계 추가** — 불일치 시 재입력 요구, 일치 시에만 role=instructor 세션 부여 | 세션에 role/group 저장 | [3-1](./조별_프로젝트_평가기준.md#3-1-평가자-구성) |
| F2 | 평가 대상 필터링 | P2 진입 | role, group | student → 전체 조 - 본인 조 / instructor → 전체 조 | 대상 조 리스트 | [4장 자기 조 채점 제외](./조별_프로젝트_평가기준.md#4-평가-시-유의사항) |
| F3 | 세부항목 6단계 선택 | P3 각 항목 버튼 탭 | 항목ID, 선택레벨(0~5) | score = 레벨/5 × 배점, 소수점 발생 시 **소수 첫째 자리에서 반올림**. **선택 즉시 Evaluation 레코드에 자동 임시저장**(isLocked=false, 제출완료 아님) → 앱 이탈 후 재진입해도 값 복원 | 항목별 점수, 실시간 합계 | [2장 평가 항목](./조별_프로젝트_평가기준.md#2-평가-항목-및-세부-기준-총-100점) |
| F4 | 자기신고 연동 표시 | P3 진입 시 | groupId | 조의 5-1 자기신고 데이터 조회 | 완료범위/향후계획/AI협업 카드 출력 | [5-1](./조별_프로젝트_평가기준.md#5-1-발표-필수-고지-항목-조별-자기신고) |
| F5 | 평가 제출 완료 처리 | P4 제출 버튼 | evaluatorId, groupId, 의견(13개 점수는 F3에서 이미 자동저장됨) | 의견 저장 + `submitted=true` 전환(최초 제출 시 `anonLabel`을 해당 조 기준 다음 순번으로 1회 배정). 마감 처리(F13) 전까지는 P2 재진입 시 기존 값을 불러와 수정 후 재제출 가능(`anonLabel`은 유지) | 제출/수정 완료 상태 | — |
| F6 | 절사 대상 산정 | 결과 조회 시점(P5/P6/I1 진입) 또는 F13 마감 처리 시 | groupId 기준 학생 평가자 점수 목록 | 총점 기준 정렬 → 최고 2인·최저 2인 플래그(5인 미만이면 절사 생략) | 평가자별 절사여부(bool) | [3-2 최종 점수 산출 방법](./조별_프로젝트_평가기준.md#3-2-최종-점수-산출-방법-절사-가중평균) |
| F7 | 점수 계산(잠정/확정) | F6 이후 자동 실행 | 절사 후 학생 점수 합, 강사 점수, 유효인원수 n-4 | `(Σ학생점수 + 강사점수×3) ÷ ((n-4)+3)` — 세부항목별로 동일 로직 반복 후 합산. **마감 전에는 조회할 때마다 재계산해 잠정치로만 반환(저장하지 않음), F13 마감 처리 시 1회만 AggregatedResult에 확정 저장** | 조별 총점 + 13개 세부항목 점수(잠정 or 확정) | [3-2 산출식](./조별_프로젝트_평가기준.md#3-2-최종-점수-산출-방법-절사-가중평균) |
| F8 | 결과 리스트 렌더링 | P5 진입/새로고침 | 전체 조 F7 결과 | 총점 내림차순 정렬, 동점 공동순위 처리 | 조 이름+총점 리스트 | [3-2 조별 순위](./조별_프로젝트_평가기준.md#3-2-최종-점수-산출-방법-절사-가중평균) |
| F9 | 조별 상세 렌더링 | P6 진입 | groupId | F7의 세부항목 점수(잠정/확정) + 해당 조 의견 전체 조회(anonLabel 적용) | 13개 항목 표 + 익명 의견 리스트 | [5-2 세부항목 채점표](./조별_프로젝트_평가기준.md#5-2-세부항목-채점표) |
| F10 | 강사 매트릭스 조회 | I1 진입 | groupId(선택) | 평가자별 원점수 + F6 절사플래그 join | 평가자×점수×절사여부 표 | [3-2](./조별_프로젝트_평가기준.md#3-2-최종-점수-산출-방법-절사-가중평균) |
| F12 | 진행 현황 갱신 | 임의 제출 발생 시 | groupId | (완료 평가자 수 / 전체 대상 평가자 수) 계산 | N/M 완료 배지 | — |
| F13 | 평가 마감 처리 | I1 "평가 마감 처리" 버튼(강사 전용, **강사가 수동으로 클릭해 실행** — 자동 마감 없음) | 없음(강사 확인 클릭) | 전체 Evaluation의 `isLocked=true` 일괄 전환 → F6·F7을 전 조·전 세부항목에 대해 1회 실행 → AggregatedResult 확정 저장(`status=FINAL`) | 확정된 조별 최종 점수, P5/P6 상태를 "확정 결과"로 전환 | [3-2 최종 점수 산출 방법](./조별_프로젝트_평가기준.md#3-2-최종-점수-산출-방법-절사-가중평균) |

## 3. 점수 산출 엔진 흐름 (핵심 로직)

```mermaid
flowchart TD
    A["평가자 제출/수정 F5<br/>(학생·강사 공통, role로 구분)"] --> B[조별로 그룹핑]
    B --> D{학생 평가자 수 ≥ 5?}
    D -->|예| E[총점 기준 정렬 후<br/>최고 2인 · 최저 2인 절사]
    D -->|아니오| F[절사 생략, 전원 반영]
    E --> G[절사 후 학생 점수 합계]
    F --> G
    G --> H[강사 점수 × 3 가산]
    H --> I["÷ (유효 학생 인원 + 3)"]
    I --> J{강사가 평가 마감<br/>처리(F13) 실행?}
    J -->|아니오| P[잠정 계산 결과 반환<br/>저장하지 않음, 조회할 때마다 재실행]
    J -->|예| Q[AggregatedResult에<br/>확정 저장 status=FINAL]
    P --> K[세부항목 13개 각각 동일 절차 반복]
    Q --> K2[세부항목 13개 각각 동일 절차 반복]
    K --> L[P5 전체 결과 잠정 표시 F8]
    K --> M[P6 조별 상세 잠정 표시 F9]
    K2 --> L2[P5 전체 결과 확정 표시 F8]
    K2 --> M2[P6 조별 상세 확정 표시 F9]
    E --> N[I1 절사여부 플래그 저장 F10]
```

> 세부항목 단위 산출은 총점과 동일한 절사·가중평균 공식을 13회 반복 적용합니다. ([3-2 평가기준 — "항목별 점수도 동일한 절사·가중 방식으로 산출"](./조별_프로젝트_평가기준.md#3-2-최종-점수-산출-방법-절사-가중평균))
> 마감 전에는 P5/P6이 "잠정 결과"를 실시간 재계산해 보여주고, 강사(이정현)가 I1에서 마감 처리(F13)를 **수동으로** 1회 실행하면(자동 마감 없음) 그 시점의 값이 "확정 결과"로 고정 저장됩니다.
> 절사·가중평균 계산 과정에서 소수점이 발생하면 **소수 첫째 자리에서 반올림**한 값을 최종 점수로 사용합니다.

## 4. 데이터 모델 개요

| 엔터티 | 주요 속성 | 비고 |
|---|---|---|
| User | id, name, role(student/instructor), groupId(학생만), passwordHash(강사만) | 명단.txt 기반 시드 데이터. 강사 레코드는 이정현 1인 고정 |
| Group | id, name, leader, members[], topic, selfReport{완료범위, 향후계획, AI협업범위} | 5-1 자기신고 포함 |
| Evaluation | id, evaluatorId, groupId, itemScores[13], comment, submitted, anonLabel, isLocked, submittedAt, lastEditedAt | 학생/강사 공통, role로 가중치 구분. `itemScores`는 P3에서 선택 즉시 자동저장(F3), `submitted`는 F5(P4 제출)에서 true로 전환. `anonLabel`은 해당 조 기준 최초 제출 순번으로 1회만 배정되고 이후 재제출해도 불변. `isLocked`는 F13 마감 처리 시에만 일괄 true로 전환(제출 시 즉시 잠기지 않음). 익명 표시(P6)는 화면 렌더링 시 `anonLabel`을 사용하고, I1에서는 실명(evaluatorId→User.name)을 그대로 노출 |
| AggregatedResult | groupId, perItemFinalScore[13], totalScore, rank, trimmedEvaluatorIds[], status(PROVISIONAL/FINAL) | 마감 전에는 저장하지 않고 조회 시 즉석 계산(PROVISIONAL), F13 실행 시 1회 저장(FINAL) |

## 6. 확정된 계산·운영 세부사항

| 항목 | 결정 사항 |
|---|---|
| F3 반올림 규칙 | 6단계 선택값(0/20/40/60/80/100%) 적용 시 소수점이 발생하면 **소수 첫째 자리에서 반올림**. (대부분의 배점은 5의 배수라 정수로 떨어지지만, 세부항목 절사·가중평균 집계 과정에서 발생하는 소수점에도 동일 규칙 적용) |
| F13 마감 시각 결정 방식 | 사전 고정 일시에 의한 자동 마감은 두지 않으며, **강사(이정현)가 I1에서 "평가 마감 처리" 버튼을 수동으로 클릭**해야 마감 처리(F13)가 실행됨 |
| 평가 참여 완결성 | **강사 미채점 상태, 학생 평가자 5인 미만 상태는 운영상 발생하지 않는 것으로 확정**(모든 학생·강사가 마감 전 채점을 완료하는 것이 운영 원칙). 따라서 F7의 강사 점수 결측 예외 처리, F6의 절사 생략(5인 미만) 분기에 대한 별도 방어 로직은 개발하지 않음 — mermaid의 해당 분기는 참고용으로만 유지 |
| 강사 채점 경로 | **P3/P4로 통일**. 강사도 F3(P3)·F5(P4)를 학생과 동일하게 사용해 채점하며, 저장 시 role=instructor로 구분되어 F7 계산 단계에서 ×3 가중치가 자동 적용됨. I1 전용 채점 입력 기능(구 F11)은 폐기 — I1은 조회·관리 전용 |

> ~~F7 재계산 시점~~과 ~~F11 강사 다수 시 처리~~는 [앱_페이지구성_내비게이션맵.md 5장 — 확정된 운영 정책](./앱_페이지구성_내비게이션맵.md#5-확정된-운영-정책)에서 각각 "마감 전 잠정치/마감 후 확정치" 구조와 "강사 이정현 1인 고정"으로 해결되어 본 목록에서 제외했습니다.
